## How Does a Model Actually Learn?

In Chapter 2, we saw that a neural network has weights — numbers that determine how strongly each connection matters. Training means finding the weight values that make the network's predictions as accurate as possible.

But how do we find those values? There are roughly 175 billion weights in GPT-3. We can't try every possible combination. We need a smarter approach — that's **gradient descent**.

This loop — forward pass, compute loss, compute gradient, update weights — runs millions of times. That's all training is. The cleverness lies in doing it efficiently for billions of weights at once, which is what backpropagation (Chapter 4) handles.

## Beyond SGD: Adaptive Optimizers

The update rule from Step 5 — `w = w − lr × gradient` — is called Stochastic Gradient Descent (SGD). It works, but it has a significant limitation: every weight uses the same learning rate, regardless of how frequently it's updated or how noisy its gradient is.

**The problem:** some weights have consistently large gradients and get over-updated; others have small, sparse gradients and barely change. A flat learning rate fits neither well.

**Adam** (Adaptive Moment Estimation, 2014) solves this by tracking two moving averages for each weight — its gradient history and its squared-gradient history — and uses them to adapt the learning rate per weight.

<MermaidDiagram chart={`graph TD
  classDef sgd fill:#fee2e2,stroke:#f87171,color:#7f1d1d
  classDef mom fill:#fef3c7,stroke:#f59e0b,color:#78350f
  classDef adam fill:#dcfce7,stroke:#4ade80,color:#14532d

  SGD["SGD  w = w − lr × ∇\n────────────────────\nSame lr for every weight\nNoisy updates\nSlow on sparse features"]
  MOM["SGD + Momentum\nw = w − lr × (β·v + ∇)\n────────────────────\nAccumulates past gradients\nSmoother descent\nLike a ball rolling downhill"]
  ADAM["Adam  (most common)\nm = β₁m + (1−β₁)∇  ← gradient mean\nv = β₂v + (1−β₂)∇²  ← gradient variance\nw = w − α · m̂ / (√v̂ + ε)\n────────────────────\nPer-weight adaptive lr\nBuilt-in momentum\nDefault: β₁=0.9 β₂=0.999 ε=1e-8"]

  SGD --> MOM --> ADAM

  class SGD sgd
  class MOM mom
  class ADAM adam`}
/>

**How Adam works in plain English:**
- `m` (first moment) is a running average of the gradient — like momentum, it smooths out noise
- `v` (second moment) is a running average of the *squared* gradient — it tracks how volatile each weight's gradient is
- The update `m̂ / √v̂` divides by the square root of recent gradient variance: weights with noisy gradients get a smaller effective step; weights with consistent gradients get a larger step
- `m̂` and `v̂` are bias-corrected versions of `m` and `v` (because they start at zero, they're biased toward zero early in training)

**Why this matters for the rest of the course:** Every chapter on fine-tuning and RLHF uses Adam or a variant (AdamW). When you see `lr=1e-4` or `lr=2e-5` as a recommended hyperparameter, that's the Adam learning rate α — not a raw SGD step size. Adam is so ubiquitous that "the optimizer" almost always means Adam unless stated otherwise.

**AdamW** adds weight decay directly into the update (rather than through the gradient), which regularizes better. It's the default for fine-tuning transformers.
