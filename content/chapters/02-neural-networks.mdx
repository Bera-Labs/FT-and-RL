## What is a Neural Network?

A neural network is a type of ML model loosely inspired by the human brain. It's made up of **neurons** organized in **layers**. Data flows in through the first layer, gets transformed at each layer, and a prediction comes out at the last layer.

There are three types of layers:

- **Input layer**: receives the raw data (e.g., pixel values of an image, or word numbers in a sentence)
- **Hidden layers**: transform the data step by step, extracting increasingly abstract features
- **Output layer**: produces the final prediction (e.g., "cat" or "dog", or a probability)

## Why Layers?

Each layer learns to detect increasingly abstract concepts. In an image recognition network:

The deeper the network (more layers), the more complex the concepts it can represent. This is why large language models like GPT have dozens to hundreds of layers.

## How Many Neurons and Layers?

This is one of the big design decisions in ML, and there's no perfect formula. Common starting points:

- **Layers**: 2–4 for simple tabular data; dozens to hundreds for images or language
- **Neurons per layer**: 64–512 is common for hidden layers; varies a lot by task
- **Output neurons**: one per class (e.g., 1000 output neurons for ImageNet's 1000 categories)

For the large pre-trained models we'll use in fine-tuning (like GPT or BERT), these decisions have already been made by the model's creators. You inherit their architecture.

## Activation Functions: The Choices Matter

ReLU is the simplest activation function but not the only one. The choice of activation meaningfully affects what a network can learn and how well it trains. Here are the functions you'll encounter most often:

**Why GeLU matters for this course:** Every Transformer-based LLM you'll work with in Parts 2–4 uses GeLU (or its variant SiLU) in its feed-forward layers instead of ReLU. GeLU applies a smooth, probabilistic gate — unlike ReLU's hard cutoff at zero, GeLU allows small negative values to partially pass through. This smoother gradient landscape helps large models train more stably.

The formula: GeLU(x) = x × Φ(x), where Φ(x) is the standard normal cumulative distribution function. In practice, a fast approximation is used: GeLU(x) ≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³))).

**Plain English for the formula:** Φ(x) is just the probability that a random number drawn from a standard bell curve is ≤ x. At x = 0 that probability is 0.5 (the midpoint of the bell). At large positive x, it's nearly 1 (almost the whole bell is to the left). So GeLU(x) = x × Φ(x) smoothly ramps from 0 (at very negative x) to x (at large positive x) — it's like a gate that opens gradually rather than snapping open at 0 the way ReLU does. The tanh approximation just computes Φ(x) quickly without a full statistical table lookup.

## Layer Normalization: Keeping Activations Stable

One more building block you'll see constantly in LLMs: **Layer Normalization**. After the activation function in each layer, modern networks apply layer norm to keep values from growing out of control.

Layer Norm computes the mean and variance of a single token's representation across its dimensions, then normalizes:
```
y = (x − μ) / (σ + ε) × γ + β
```
where γ and β are learnable parameters (scale and shift).

**Plain English for the formula:** (x − μ) / σ is just subtracting the average and dividing by the spread — the same z-score calculation you'd use to standardize exam scores so the class average is 0 and one standard deviation is 1. After that rescaling, every layer receives inputs in a consistent, well-behaved range regardless of how wild the raw values were. γ and β then let the network learn to re-scale and re-center if a particular layer works better with a different range — they're the 'undo' knobs that give the model flexibility. The ε is a tiny number (like 1e-5) added to σ to prevent a divide-by-zero if all values in a layer happen to be identical.

This is different from Batch Normalization (which normalizes across the batch dimension). Transformers use Layer Norm because it works on single sequences without needing batch statistics — important for variable-length text.

**Why it matters:** Without normalization, activations in deep networks drift to very large or very small values as the signal passes through many layers. Layer Norm keeps activations in a healthy range, which makes gradients flow better and training more stable. You'll see `layer_norm` mentioned throughout the chapters on Transformers and fine-tuning — this is what it is.
