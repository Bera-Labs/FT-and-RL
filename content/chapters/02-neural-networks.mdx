## What is a Neural Network?

A neural network is a type of ML model loosely inspired by the human brain. It's made up of **neurons** organized in **layers**. Data flows in through the first layer, gets transformed at each layer, and a prediction comes out at the last layer.

There are three types of layers:

- **Input layer**: receives the raw data (e.g., pixel values of an image, or word numbers in a sentence)
- **Hidden layers**: transform the data step by step, extracting increasingly abstract features
- **Output layer**: produces the final prediction (e.g., "cat" or "dog", or a probability)

The deeper the network (more layers), the more complex the concepts it can represent. This is why large language models like GPT have dozens to hundreds of layers.

## How Many Neurons and Layers?

This is one of the big design decisions in ML, and there's no perfect formula. Common starting points:

- **Layers**: 2вЂ“4 for simple tabular data; dozens to hundreds for images or language
- **Neurons per layer**: 64вЂ“512 is common for hidden layers; varies a lot by task
- **Output neurons**: one per class (e.g., 1000 output neurons for ImageNet's 1000 categories)

For the large pre-trained models we'll use in fine-tuning (like GPT or BERT), these decisions have already been made by the model's creators. You inherit their architecture.

**Why GeLU matters for this course:** Every Transformer-based LLM you'll work with in Parts 2вЂ“4 uses GeLU (or its variant SiLU) in its feed-forward layers instead of ReLU. GeLU applies a smooth, probabilistic gate вЂ” unlike ReLU's hard cutoff at zero, GeLU allows small negative values to partially pass through. This smoother gradient landscape helps large models train more stably.

## Layer Normalization: Keeping Activations Stable

One more building block you'll see constantly in LLMs: **Layer Normalization**. After the activation function in each layer, modern networks apply layer norm to keep values from growing out of control.

Layer Norm computes the mean and variance of a single token's representation across its dimensions, then normalizes:
```
y = (x в€’ Ој) / (Пѓ + Оµ) Г— Оі + ОІ
```
where Оі and ОІ are learnable parameters (scale and shift).

This is different from Batch Normalization (which normalizes across the batch dimension). Transformers use Layer Norm because it works on single sequences without needing batch statistics вЂ” important for variable-length text.

**Why it matters:** Without normalization, activations in deep networks drift to very large or very small values as the signal passes through many layers. Layer Norm keeps activations in a healthy range, which makes gradients flow better and training more stable. You'll see `layer_norm` mentioned throughout the chapters on Transformers and fine-tuning вЂ” this is what it is.
