## The Scale Problem

Tabular Q-learning hits a wall with large state spaces. DQN (Deep Q-Network, Mnih et al., 2013) solved this by replacing the Q-table with a deep neural network.

## Prioritized Experience Replay

Standard experience replay samples transitions uniformly at random — every stored experience is equally likely to be selected. This is wasteful: if the agent has already learned that "walking into a wall does nothing," replaying that transition for the hundredth time teaches it nothing new. Meanwhile, the one transition where it first discovered a bonus room might never get resampled.

**Prioritized Experience Replay (PER)** fixes this by sampling the transitions where the model is most wrong — measured by how far off its Q-value prediction was (the TD error). Transitions where the model is already accurate get sampled rarely; transitions where the model is still badly wrong get sampled often. This focuses learning where it's most needed.

PER assigns a priority to each stored transition and samples high-priority transitions more often. The priority is based on the TD error |δ| = |r + γ max Q(s') − Q(s, a)| — transitions where the model is most wrong get the highest priority.

  class UNIF mid

**How it works:**

1. When a transition is stored, assign priority p_i = |δ_i| + ε (ε prevents zero-priority transitions from never being sampled)
2. Sampling probability: P(i) = p_i^α / Σ p_j^α — the exponent α controls how aggressively to prioritize (α=0 → uniform, α=1 → fully proportional)
3. After computing the TD error for a sampled transition, update its priority to the new |δ_i|
4. Apply importance-sampling weights to correct the gradient bias introduced by non-uniform sampling

**Implementation:** a sum tree data structure (binary tree where parent = sum of children) makes priority-proportional sampling and priority updates O(log N) instead of O(N). This is what makes PER practical for buffers of 1 million transitions.

**Gain:** PER typically improves DQN performance by ~20–30% on Atari games and can reduce the number of environment interactions needed by 2–3×. It's part of the Rainbow DQN combination.

## DQN's Legacy

DQN was a landmark paper — it demonstrated that a single neural network architecture, trained with RL, could reach human-level performance on dozens of games from raw pixels, with no game-specific engineering. It launched modern deep RL.
