## What is Backpropagation?

In Chapter 3, we said that gradient descent needs the gradient — a number for every weight that says "changing this weight by a tiny amount changes the loss by this much." But we glossed over a key question: **how do you actually compute those gradients** when a network has billions of weights spread across dozens of layers?

Backpropagation is the answer. It's an efficient algorithm that applies the **chain rule of calculus** backwards through the network, computing every weight's gradient in one pass.

## Why This Works at Scale

Backprop computes every gradient in just **two passes** through the network — one forward, one backward. Compare this to naively estimating each gradient by perturbing one weight at a time: for GPT-3's 175 billion weights, that would require 175 billion forward passes per training step. Backprop does it in 2.

This efficiency is what made deep learning practical. Every major neural network — image classifiers, language models, diffusion models — is trained using backpropagation.

## Vanishing and Exploding Gradients

Backpropagation multiplies gradients together as it moves backwards through layers. This multiplication can cause two catastrophic failures in deep networks:

**Vanishing gradients:** If the gradient at each layer is a small number (say 0.1), then after 30 layers the gradient reaching the first layer is 0.1³⁰ ≈ 10⁻³⁰ — effectively zero. The early layers stop learning because they receive no signal.

**Exploding gradients:** If the gradient at each layer is large (say 2.0), after 30 layers it becomes 2³⁰ ≈ 10⁹ — the weights receive enormous updates and the network destabilizes. In practice this shows up as NaN (not-a-number) losses.

### Four solutions that modern networks use

**1. Gradient clipping** — the simplest fix for exploding gradients. Before applying the update, check the total gradient norm. If it exceeds a threshold (e.g. 1.0), scale all gradients down proportionally:
```
if ‖∇‖ > max_norm:
    ∇ = ∇ × max_norm / ‖∇‖
```
This prevents any single update from being catastrophically large. Gradient clipping is the default in all modern training frameworks and is why you often see `max_grad_norm=1.0` in fine-tuning configs.

**2. Residual connections (skip connections)** — instead of computing a new representation from scratch, each layer *adds its output to its input*:
```
output = layer(x) + x
```
The `+ x` term creates a direct highway for gradients. Even if the `layer(x)` gradients vanish, gradients still flow straight through via the `+ x` path. This is why ResNets (image models) and Transformers can have 96–200+ layers — residual connections make deep networks trainable.

**3. Layer Normalization** — normalizes activations within each layer to have zero mean and unit variance, preventing activations from growing extremely large or small as they pass through many layers. You saw this in Chapter 2; this is the reason it exists.

**4. Better initialization** — choosing starting weights carefully (e.g. Xavier/Glorot, He initialization) keeps the variance of activations roughly constant across layers, preventing the signal from amplifying or collapsing before training even starts.

This is why the Transformer architecture (Chapter 7) has a very specific structure: attention → add & layer norm → FFN → add & layer norm. The "add" is the residual connection and the "layer norm" prevents activation drift. Both are direct solutions to the vanishing gradient problem that plagued earlier deep networks.
