## Why DPO Was Invented

PPO-RLHF is powerful but expensive. Aligning a single large language model requires four neural networks running simultaneously: the policy being trained, a frozen reference model, a reward model, and a value network. DPO achieves the same alignment goal with only two models.

<ConceptCheck
  question="In the DPO loss, what does the reference model π_ref actually do? What would happen if you removed it and just trained to increase P(y_w) and decrease P(y_l) directly?"
  answer="The reference model acts as an anchor. It measures how likely the reference SFT policy would have generated each response, and the DPO loss maximizes the margin of the policy's log-ratio above the reference's log-ratio — not the raw log-probability. Without the reference model, the policy could minimize loss by collapsing: assigning near-zero probability to both chosen and rejected responses makes both log-probs very negative, and the margin between them could still appear favorable. The reference model prevents this collapse by anchoring the policy close to the SFT distribution. Variants like ORPO and SimPO replace the reference model with alternative regularization (an explicit SFT loss term, or length normalization) to achieve the same stabilizing effect without running a second model."
/>
