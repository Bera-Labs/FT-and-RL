## The Core Idea

LoRA (Low-Rank Adaptation, Hu et al., 2021) rests on one observation: when you fine-tune a model on a specific task, the changes to the weight matrices have **low intrinsic rank**. This means the update ΔW doesn't need all its dimensions — it can be well-approximated by two small matrices multiplied together.

<MermaidDiagram chart={`graph LR
  classDef frozen fill:#dbeafe,stroke:#60a5fa,color:#1e3a8a
  classDef lora fill:#dcfce7,stroke:#4ade80,color:#14532d
  classDef math fill:#6366f1,stroke:#4f46e5,color:#fff

  W["W (frozen)\n4096 × 4096\n= 16,777,216 params\n🔒 never updated"] --> OUT
  A["A matrix ✏️\n4096 × r\nr = 16\n= 65,536 params"] --> B["B matrix ✏️\nr × 4096\nr = 16\n= 65,536 params"]
  B --> DELTA["ΔW = B × A\n(low-rank approximation\nof the full update)"]
  DELTA --> OUT["Output = Wx + ΔWx\n= Wx + BAx"]
  W --> OUT

  class W frozen
  class A,B lora
  class DELTA,OUT math`}
/>

Instead of 16.7M trainable parameters per weight matrix, LoRA uses just 131k — less than **0.8%** — and achieves results that are remarkably close to full fine-tuning.
