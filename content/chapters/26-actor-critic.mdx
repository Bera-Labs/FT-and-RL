## Combining Value-Based and Policy-Based RL

Actor-Critic methods unify the strengths of both approaches:
- **Policy gradient** (actor): learns the policy directly — works with any action space
- **Value function** (critic): estimates how good each state is — provides low-variance feedback

## N-Step Returns: Bridging TD and Monte Carlo

The one-step advantage from Step 3 (A = r + γV(s') − V(s)) bootstraps off a single step. The full-return advantage from REINFORCE uses all future rewards with no bootstrapping. **N-step returns** sit between these extremes, and understanding them clarifies what GAE is actually computing.

An n-step return uses the actual rewards for the next n steps, then bootstraps from the value estimate at step n:

```
G_t^(1)  =  r_t + γ V(s_{t+1})                         (1-step TD)
G_t^(2)  =  r_t + γ r_{t+1} + γ² V(s_{t+2})            (2-step return)
G_t^(n)  =  r_t + γ r_{t+1} + ... + γⁿ V(s_{t+n})      (n-step return)
G_t^(∞)  =  r_t + γ r_{t+1} + γ² r_{t+2} + ...         (Monte Carlo)
```

**Why reward propagation matters:** With a 1-step advantage, a reward discovered 10 steps in the future takes 10 training iterations to propagate back to the actions that caused it. With n-step returns or GAE (λ close to 1), that reward propagates much faster — the agent learns more quickly that earlier actions led to the eventual reward.

**In RLHF context:** language model generation is a single episode where the reward arrives only at the end (after the full response is scored). PPO uses GAE with λ=0.95 to propagate this end-of-response reward backward through all 200+ token generation steps efficiently.
